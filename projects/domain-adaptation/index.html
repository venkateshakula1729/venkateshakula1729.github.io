Continual Domain Adaptation

Jan – Apr 2024IIT KanpurGitHub ↗
[ Hero Image — Distribution shift across domains D₁ to D₂₀ ]

Overview

Real-world data is non-stationary — the distribution that a model was trained on drifts over time. This project addressed continual domain adaptation: how to sequentially adapt a classifier across 20 domains while retaining performance on all previously seen domains, without storing historical data.

Motivation

Standard supervised learning assumes training and test data come from the same distribution. In production, this assumption breaks down constantly — user behavior shifts, sensor calibration drifts, market regimes change. The naive approach (retraining from scratch) is expensive. Fine-tuning on new data causes catastrophic forgetting of old domains. The challenge is to adapt continuously while remembering.

Technical Details

LwP Classifier with Gaussian Prototypes

Instead of storing raw historical data, we represent each class-domain combination as a Gaussian prototype — a mean vector and covariance matrix in feature space. Classification is then nearest-prototype assignment. This gives us a compact, privacy-friendly summary of past domains.

Sequential Adaptation Strategy

For a sequence of 20 domain-specific classifiers {f₁, f₂, ..., f₂₀}:

  • Parameter blending — new model parameters are a weighted average of the previous model and the domain-specific update, with weights determined by domain similarity
  • Prototype replay — synthetic examples generated from stored Gaussian prototypes are mixed with new domain data during training
  • SWD minimization — Sliced Wasserstein Distance between source and target feature distributions guides the adaptation objective

On homogeneous domains (D₁–D₂₀ with similar feature distributions), the framework maintained 88% accuracy. On heterogeneous distributions (D₁₁–D₂₀ with significant shift), accuracy ranged from 71% to 87%, with catastrophic forgetting limited to 0.03%.

Lessons Learned

  • Gaussian prototypes are a surprisingly effective proxy for historical data — compact, differentiable, and generatively useful
  • SWD is more practical than full Wasserstein distance for high-dimensional alignment — the slicing provides computational tractability without sacrificing too much fidelity
  • The forgetting-adaptation tradeoff is the central challenge — more aggressive adaptation to new domains inevitably comes at the cost of old domain performance

Future Work

  • Extending to non-Gaussian prototype representations (mixture models, normalizing flows)
  • Online adaptation without explicit domain boundaries — detecting shift from the data stream itself
  • Application to financial time-series where regime changes are analogous to domain shifts
