Adobe Research · Big Data Experience Lab

Multi-Agent LLM Workflow Optimization

May – Jul 2025 Intern Research Assistant Multi-Agent Systems · LLMs · Prompt Optimization

Overview

During my research internship at Adobe's Big Data Experience Lab, I investigated a fundamental problem in multi-agent LLM systems: as the number of agents increases, communication overhead grows superlinearly, leading to token waste, context pollution, and latency degradation. The goal was to design communication protocols that preserve task performance while dramatically reducing computational cost.

Motivation

Multi-agent LLM architectures are increasingly deployed for complex tasks — code generation, multi-hop reasoning, debate-based verification — where a single model falls short. However, existing frameworks like AutoGen and LangChain treat inter-agent communication as unconstrained text exchange. This leads to several problems:

  • Token explosion — agents repeat context unnecessarily across turns
  • Context window saturation — long conversations push agents past their effective context limits
  • Latency compounding — each redundant token increases end-to-end response time
  • Quality degradation — irrelevant context introduces noise into downstream reasoning

Technical Details

Framework Evaluation

I began by systematically benchmarking existing agent communication frameworks — AutoGen, LangChain — and prompt optimization systems — DSPy, TextGrad, AdalFlow — across standardized tasks. This evaluation revealed consistent patterns: most frameworks optimize for individual agent quality but neglect inter-agent communication efficiency.

Context-Aware Communication Protocol

The core contribution was a task-aware communication protocol that enables agents to share only the information relevant to the current subtask. The protocol includes:

  • A context compression layer that identifies and removes redundant information before inter-agent transfer
  • A task-decomposition module that routes specific context slices to the agents that need them
  • A token-efficient training pipeline that maintains accuracy while operating on compressed representations

Benchmark Suites

Validated on HotpotQA (multi-hop QA), GPQA (graduate-level reasoning), and GSM8K (mathematical problem-solving) in multi-turn, long-context settings. The system consistently outperformed state-of-the-art baselines on both efficiency and accuracy metrics.

Lessons Learned

  • Communication overhead is often the dominant bottleneck in multi-agent systems — not individual agent quality
  • Prompt optimization frameworks (DSPy, TextGrad) are powerful but orthogonal to the communication efficiency problem
  • Context compression must be task-aware; naive truncation destroys critical reasoning chains
  • Real-time coordination is feasible even with large language models when communication is structured

Future Work

  • Extending the protocol to support dynamic agent addition/removal during task execution
  • Investigating the protocol's behavior with heterogeneous model sizes (e.g., GPT-4 + smaller specialist models)
  • Formal analysis of communication complexity bounds in multi-agent LLM systems
