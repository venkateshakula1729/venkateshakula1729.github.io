Stock Forecasting with Deep Learning

Mar – Apr 2024IIT KanpurGitHub ↗
[ Hero Image — Time-series prediction overlay on candlestick chart ]

Overview

This project explored temporal deep learning for financial time-series prediction. Using LSTM networks — designed specifically for capturing long-range dependencies in sequential data — I built a model that predicts closing stock prices for the next 2 trading days given a 50-day lookback window, trained on a decade of historical market data.

Motivation

Financial time-series exhibit complex non-linear patterns — trend, seasonality, volatility clustering, and regime changes. Traditional statistical models (ARIMA, GARCH) make strong distributional assumptions that often fail in practice. LSTMs can learn these patterns directly from data without explicit feature engineering, making them a natural choice for this task.

The project also served as a practical exercise in the challenges specific to financial ML: lookahead bias, non-stationarity, and the gap between statistical accuracy and trading profitability.

Technical Details

Data Preprocessing

  • Normalization and scaling — MinMax scaling to [0,1] range, fit only on training data to prevent lookahead bias
  • Outlier clipping — Winsorization at the 1st/99th percentiles to reduce the impact of flash crashes and gaps
  • Missing value handling — forward-fill for weekends and holidays, with volume-weighted interpolation for intraday gaps
  • Sliding window construction — 50-day input sequences mapped to 2-day output targets

LSTM Architecture

A multi-layer LSTM with dropout regularization, trained with Adam optimizer and MSE loss. The architecture was deliberately kept simple to test whether temporal modeling alone — without auxiliary features like technical indicators — can capture meaningful price dynamics.

Lessons Learned

  • Directional accuracy (95.3%) is more practically meaningful than RMSE for trading applications — knowing which way the price moves matters more than the exact magnitude
  • Simple LSTM architectures can be surprisingly effective when preprocessing is done correctly — feature engineering matters more than model complexity
  • Financial ML requires extreme care with data leakage — even subtle lookahead bias can inflate metrics dramatically
  • The gap between prediction accuracy and profitable trading is significant — transaction costs, slippage, and market impact are not captured by RMSE

Future Work

  • Incorporating attention mechanisms (Temporal Fusion Transformer) for interpretable multi-horizon forecasting
  • Adding auxiliary features: options-implied volatility, order flow imbalance, macro indicators
  • Backtesting with realistic transaction costs and position sizing to evaluate economic significance
  • Regime detection to switch between model variants in different market conditions
