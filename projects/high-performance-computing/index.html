High-Performance Computing with MPI

May – Jun 2025IIT KanpurGitHub ↗
[ Hero Image — 3D domain decomposition grid ]

Overview

This project tackled a core challenge in scientific computing: efficiently computing extrema values over large 3D time-series datasets using distributed memory parallelism via MPI. The focus was on optimal domain decomposition, eliminating I/O bottlenecks, and achieving near-linear speedup on a multi-core cluster.

Motivation

Scientific simulations routinely produce 3D volumetric data over time — weather models, fluid dynamics, molecular simulations. Analyzing these datasets sequentially becomes infeasible as grid resolution increases. The challenge is to decompose both the computation and the I/O across multiple processes while minimizing communication overhead.

Technical Details

3D Domain Decomposition

The 3D dataset (up to 64 × 64 × 64 dimensions) was decomposed across MPI processes using a balanced partitioning strategy. Each process handles a contiguous sub-volume, with ghost cells for boundary communication.

Parallel I/O Optimization

The initial implementation used sequential I/O — a single process read the entire dataset and scattered it to others. This was the dominant bottleneck. I replaced it with custom MPI subarray datatypes that allow each process to independently read its sub-volume directly from disk using MPI-IO, achieving 8× faster data loading.

Communication Pattern

Boundary exchange between neighboring subdomains used non-blocking MPI sends/receives (MPI_Isend/MPI_Irecv) to overlap communication with computation. The extrema reduction used MPI_Allreduce with custom operators.

Lessons Learned

  • I/O is almost always the bottleneck in HPC — optimizing computation without fixing I/O yields marginal gains
  • MPI subarray datatypes are underused but powerful — they eliminate the need for manual buffer packing/unpacking
  • Non-blocking communication with computation overlap is essential for scaling beyond 8 processes
  • Strong scaling efficiency drops off due to communication-to-computation ratio as subdomain size decreases

Future Work

  • Hybrid MPI + OpenMP parallelism for intra-node optimization
  • Adaptive mesh refinement for non-uniform data distributions
  • GPU offloading via CUDA-aware MPI for compute-intensive kernels
