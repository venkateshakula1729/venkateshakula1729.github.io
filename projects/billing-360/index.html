Billing 360: Full-Stack Retail Management

Jan – Apr 2024IIT KanpurGitHub ↗

Introduction

Developed during my fourth semester at IIT Kanpur under the mentorship of Prof. Indranil Saha, Billing 360 is an end-to-end MERN stack web application built for micro-enterprises. The project served as my first major foray into large-scale collaborative software engineering, where I worked alongside a 10-person team following rigorous Agile methodologies.

The software streamlines daily retail operations, offering a robust suite of tools including automated invoice generation, real-time inventory tracking, credit management, and data analytics. We handled the entire software development lifecycle—from drafting the initial Software Requirements Specification (SRS) and system architecture to final deployment and load testing.

Core Contributions

System Architecture & Database Design

I helped architect the core MongoDB schemas mapping to invoices, inventory states, and transactional ledgers. A primary focus was ensuring referential integrity and strict data consistency across highly coupled entities.

Invoice Engine & API Development

Collaborating closely with a sub-team, I built out the primary billing infrastructure. This required engineering several critical REST APIs responsible for PDF invoice rendering, asynchronous inventory updates, and transaction logging. On the frontend, I implemented a complex, highly responsive searchable dropdown component for rapid item entry during checkout.

UI/UX Design

I co-designed the complete frontend layout. The goal was to produce a clean, intuitive interface tailored for non-technical retail users, emphasizing responsive design and frictionless navigation.

Performance Testing & QA

Recognizing that PDF generation was a major bottleneck, I spearheaded the performance profiling for the invoice generation APIs. I also ran extensive testing phases, identifying and resolving critical concurrency bugs to prevent race conditions during simultaneous inventory updates.

Reflections

Building Billing 360 reinforced that effective software engineering extends far beyond writing code. The project was a masterclass in requirement elicitation, API design, and the realities of distributed teamwork. Gaining hands-on experience with Agile sprints, systematic bug tracking, and performance tuning fundamentally shifted my perspective on what it takes to ship production-ready full-stack applications.
